# CS2: Operational Logic & Service Guide

เอกสารนี้กำหนดมาตรฐานการจัดการข้อมูลและการคำนวณรายได้พนักงาน เพื่อความแม่นยำและง่ายต่อการดูแลรักษา

## 1. การจัดการรายได้ (Income & Currency)
ระบบเน้นความซับซ้อนน้อยที่สุด (Minimum Complexity):
- **Base Currency**: คำนวณผลงานเป็น **THB** (บาท) ตามสถิติ Views/Followers
- **Payout Currency**: แปลงยอดสุทธิเป็น **USD** สำหรับการโอนผ่าน Exchange (ห้ามบันทึกผลการคำนวณผิดพลาดแม้แต่เซนต์เดียว)
- **No Overheads**: ไม่มีระบบภาษี, ไม่มีโอที, ไม่มีประกันสังคม

## 2. การนับวันทำงาน (Working Days)
- นับวันทำงานจาก **Daily Logs** ที่พนักงานส่งเข้ามา
- แยกสถานะ "วันทำงานปกติ" และ "วันทำงานหยุด" (Double Pay 2X) ให้ชัดเจนในปฏิทิน

## 3. กฎการแยกส่วน (Separation of Concerns)
เพื่อให้ Logic ไม่ปนกับ UI:
- **Service Layer**: การคำนวณทั้งหมดต้องอยู่ใน `/src/services` หรือ `/src/lib` ห้ามเขียน Logic คำนวณในไฟล์ `.tsx`
- **Frontend Layer**: ไฟล์ UI ต้องหน้าที่แสดงผลและรับ Input เท่านั้น มีขนาดไฟล์ที่กระชับ ไม่ซับซ้อน
- **Naming Convention**: ไฟล์คำนวณควรลงท้ายด้วย `...Service.ts` หรือ `...Utils.ts`

## 4. Timezone & Audit Trail
- ทุกการบันทึกเวลาต้องใช้ **UTC (ISO 8601)**
- ต้องมีการเก็บ `source` (API/Manual) และ `staffId` ผู้ที่บันทึกข้อมูลเสมอ เพื่อใช้ในการตรวจสอบ (Audit Trail) ในกรณีที่ตัวเลขไม่ตรงกัน
