# Organization RBAC Architecture (Role-Based Access Control)

เอกสารฉบับนี้อธิบายโครงสร้างสิทธิ์การเข้าถึงข้อมูล (Data Isolation & Visibility) ของระบบ Meta HR Analytics แบ่งออกเป็น 4 ระดับ (Roles) เพื่อความปลอดภัยของข้อมูลเงินเดือนและการบริหารงานที่รัดกุม

## 👥 โครงสร้างองค์กร (Total: 17 Personnel)
พนักงานระดับปฏิบัติการ (พนักงานทำเพจ) และ ผู้บริหารระดับสูง (C-Level) 2 คน ระดับ admin, manager, staff จะนับเป็นพนักงานทั้งหมด
1. **Super Admin** ผู้บริหารระดับสูง C-Level / HR Head 
2. **Admin** พนักงานทำเพจ ค้ำจุนบทบาทผู้ช่วยดูแลระบบ Operations 
3. **Manager** พนักงานทำเพจ ค้ำจุนบทบาทหัวหน้าทีม Team Leader 
4. **Staff** พนักงานทำเพจทั่วไป 

---

## 🔐 รายละเอียดสิทธิ์แต่ละ Role (Permissions & Visibility)

### 1. Super Admin (God Mode)
มีอำนาจสิทธิ์ขาดสูงสุดในระบบ สามารถเข้าถึงและแก้ไขข้อมูลทุกเส้น
- **Visibility (การมองเห็น):** มองเห็น Dashboard ของทุกคนในบริษัท (HQ Dashboard) มองเห็นอัตราค่าคอมมิชชั่น และตัวเลขเงินเดือนรายบุคคลของทุกคน
- **Configurations:** มีสิทธิ์เข้าหน้าเมนู `Payroll & Policies` เพื่อปรับสูตรคำนวณเงินเดือน, เป้าหมายขั้นต่ำ (Min View Target), และจำนวนเงินโบนัส/บทลงโทษ
- **Actions:** 
  - สั่งทำจ่ายเงินเดือน
  - ติดป้ายสั่งงาน/แจ้งเตือน (Request/Flag) ไปยังเพจที่มีปัญหา เพื่อให้ Staff เจ้าของเพจกดรับทราบและดำเนินการแก้ไข

### 2. Admin (Master Ops)
ผู้ช่วยดูแลภาพรวมของบริษัท เน้นหนักไปที่ด้าน Operations และการทำธุรกรรม (Transactions)
- **Visibility (การมองเห็น):** มองเห็น Dashboard ของทุกคนในบริษัท (HQ Dashboard) มองเห็นตัวเลข **"สรุปยอดเงินเพื่อทำจ่าย"** แต่ไม่สามารถเจาะลึกไปแก้สูตรคำนวณได้
- **Configurations:** ❌ ไม่มีสิทธิ์แก้ไขกฎเกณฑ์ทางเงินเดือน (Payroll Policy)
- **Actions:**
  - ยืนยันการทำจ่ายเงิน (Mark as Paid)
  - เข้าถึงหน้าจัดการ Page & Account (Setup)
  - ช่วย Super Admin ติดป้ายสั่งงาน/แจ้งเตือน (Request/Flag) ไปยังเพจของ Staff ได้

### 3. Manager (Team Lead)
ผู้นำทีม (สมมติว่าดูแลทีม A หรือทีม B) เน้นการกระตุ้นผลงานและพัฒนาศักยภาพลูกทีม
- **Visibility (การมองเห็น):** มองเห็น **เฉพาะลูกทีมของตัวเองเท่านั้น** ฝั่งระบบหลังบ้าน (Service Layer) จะเช็ค `team_id` และตัดข้อมูลทีมอื่นทิ้งทั้งหมด
- **Data Stripping (การตัดข้อมูลการเงิน):** ทันทีที่ระบบรู้ว่า Role คือ Manager ระบบจะสั่งลบข้อมูล (Strip Data) ที่เกี่ยวกับ "จำนวนเงิน (Money), โบนัส (Bonus), และค่าปรับ (Penalty)" ออกจาก Payload 100% ทำให้ Manager **ไม่มีวัน** เห็นเงินเดือนของลูกทีม 
- **Actions:**
  - ดู Leaderboard การแข่งขัน (แข่งเป้าหมายวิว) ภายในทีม
  - สังเกตการณ์สุขภาพเพจ (Portfolio Health) ของลูกทีม เพื่อเทรนนิ่งหรือแนะนำเทคนิคเพิ่มเติม
  - ❌ ไม่สามารถเกี่ยวโยงกับเงินเดือนลูกทีมใดๆ ได้ทั้งสิ้น

### 4. Staff (User / Creator)
พนักงานผู้สร้างสรรค์คลิปวิดีโอและดูแลเพจต่างๆ
- **Visibility (การมองเห็น):** มองเห็น **เฉพาะ Dashboard ของตัวเอง** เท่านั้น (1 คน อาจถือหลายเพจ) ❌ ไม่เห็นยอดวิว หรือเงินเดือนของพนักงานคนอื่นเลย
- **Configurations:** ❌ ไม่มีสิทธิ์
- **Actions:**
  - กรอกส่งงานรายวัน
  - ดูยอดคำนวณเงินเดือน/โบนัสของตัวเองแบบ Real-time
  - **Acknowledge System:** หากหน้า Dashboard ตัวเองมีแถบสีแดงแจ้งเตือนจาก Admin (ว่าเพจไหนมีปัญหา) ต้องกดปุ่ม **[ รับทราบ / ยืนยันการแก้ไข ]** เพื่อรายงานผลการทำงานกลับไปยัง HQ Dashboard

---

## ⚙️ Security Layer Note (RLS Simulation)
โครงสร้างนี้ถูกเขียนดักไว้ตั้งแต่ในระดับ **Service/Database Layer** (ไฟล์ `hqDashboardService.ts` หรือ Supabase RLS Policy) 
กล่าวคือ ระบบไม่ได้เอาข้อมูลส่วนตัวไปซ่อน (Hide) ไว้ที่หน้า React Component แต่ระบบ **ไม่ได้ดึงข้อมูล** เหล่านั้นส่งออกเซิฟเวอร์ตั้งแต่แรก (Data Never Leaves Server) ทำให้สถาปัตยกรรมนี้ทนทานต่อการ Hack (Bulletproof) รัดกุมในแง่ของกฎหมายแรงงาน และปกป้องความเป็นส่วนตัวของพนักงานทุกคน
